Page 1 MATIS Feature Flags Assessment MATIS Feature Flags Migration Assessment Executive Summary The MATIS application currently manages 56 feature flags through Web.config files, creating significant challenges in CI/CD pipelines and deployment processes. This assessment evaluates the migration to Azure App Configuration, a cloud-native feature management solution that addresses critical limitations in the current approach. Key Findings: •  Current State:  56 feature flags stored in Web.config across multiple environments •  Primary Issue:  Deployments required for any feature flag changes •  Risk Level:  High - Configuration changes trigger full application restarts •  Recommendation:  Migrate to Azure App Configuration with Feature Manager •  Expected ROI:  Significant reduction in deployment overhead for feature toggles

Page 3 MATIS Feature Flags Assessment Problems with Current Approach 1. CI/CD Pipeline Challenges 1. Deployment Required for Flag Changes  [HIGH] Every feature flag toggle requires a full deployment cycle, including build, test, and release pipeline execution. 2. Environment-Specific Configuration Management  [HIGH] Maintaining separate Web.config files or transformation files for Dev, QA, UAT, and Production environments. Configuration drift is common. 3. Application Restart Requirement  [CRITICAL] Changing a feature flag requires application restart, causing downtime and disrupting active user sessions. 4. No Runtime Toggle Capability  [HIGH] Cannot quickly disable problematic features in production without a deployment. Emergency rollbacks require full deployment. 5. Limited Audit Trail  [MEDIUM] Configuration changes tracked only through source control commits. No built-in auditing of who changed what and when in runtime. 2. Deployment-Specific Problems •  Configuration Coupling:  Feature flags tightly coupled with application deployment lifecycle •  Rollback Complexity:  Rolling back a bad feature requires rolling back entire application •  Multi-Region Challenges:  Difficult to synchronize feature flags across multiple deployment regions •  Gradual Rollout Impossible:  Cannot enable features for subset of users (A/B testing, canary releases) •  Testing Overhead:  Testing different flag combinations requires multiple deployment configurations •  Emergency Response:  Slow response time for production incidents requiring feature disable

Page 4 MATIS Feature Flags Assessment 3. Operational Impact The current approach creates operational bottlenecks and increases time-to-resolution for configuration changes. Feature flag modifications require going through full deployment cycles instead of immediate updates. During incidents, delays can significantly impact business operations and user experience. Azure App Configuration Solution Azure App Configuration is a managed service that provides centralized configuration and feature management. It decouples configuration from code, enabling runtime changes without redeployment. 1. Core Capabilities Feature Management:  Built-in feature flag management with Feature Manager library integration for .NET Dynamic Configuration:  Real-time configuration updates without application restart using configuration refresh Environment Labels:  Single store for all environments with label-based filtering (Dev, QA, UAT, Production) Feature Filters:  Percentage rollouts, targeting rules, time windows, and custom filters Point-in-Time Recovery:  Configuration snapshots and versioning with 30-day retention Integration:  Seamless integration with Azure Key Vault for secrets, Azure DevOps, and GitHub Actions Audit & Monitoring:  Complete audit trail, Azure Monitor integration, and change notifications High Availability:  99.9% SLA with geo-replication support for multi-region deployments 2. Proposed Architecture •  Azure App Configuration Store:  Central repository for all feature flags and configuration •  Feature Manager Library:  Microsoft.FeatureManagement.AspNetCore NuGet package •  Configuration Provider:  Azure App Configuration provider with automatic refresh •  Updated IFeatureFlags:  Maintain existing interface, change implementation to use Feature Manager •  Frontend Updates:  Update AppConfigController to use Feature Manager service •  Managed Identity:  Secure authentication using Azure Managed Identity (no connection strings)
