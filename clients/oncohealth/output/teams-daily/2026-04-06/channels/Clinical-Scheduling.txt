After yesterday's meeting, I feel we need to carry on with investigation, at the moment we have 3 strong candidate to migrate data form Databricks to NewUM Postgress:

Lakebase Synced Tables
Databricks-native managed sync from Delta/Iceberg directly into a Lakebase-hosted PostgreSQL-compatible instance — no pipeline code to write
ADLS Gen2 Staging + Airflow + PostgreSQL COPY
Most mature and understood option - Databricks writes Parquet to ADLS, PostgreSQL bulk-loads via COPY, shadow table swap ensures zero reader disruption
Apache Iceberg → PostgreSQL
Most architecturally elegant - Databricks writes Iceberg natively (one config change), pg_lake lets PostgreSQL query it directly with no ingestion step, queries run 10x+ faster via DuckDB, Eliminates the COPY pipeline, the Airflow DAG, and the swap pattern entirely; object storage costs ~10x less per GB than Postgres block storage

More details you can find here NewUM Data System Design.docx

Data exchange mechanism – work in progress chapter

We need to investigate more over upcoming sprint those 3 approaches.
