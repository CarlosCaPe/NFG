NewUM Version 1.0 Product Scope

Summary
NewUM Version 1.0 establishes the foundational platform for OncoHealth to deliver fully
delegated chemotherapy utilization management services
in Mainland across C ommercial and Medicare lines of business. It achieves functional parity with
existing OneUM platforms while introducing meaningful improvements in user
experience, implementation eﬃciency, and clinical workﬂow and decision making. V1.0
enables OH teams to reliably intake, review, determine, and communicate decisions for
chemotherapy treatment requests to health plans, providers, and members—within
compliance timeframes— using a standardized, rules driven platform.
This is not the ﬁnal version intended for a live health plan launch. Instead, V1.0 delivers a broad
enough scope to stand up a health plan in a production- like environment and
conduct extensive user testing with complex cases— a critical step to derisk the ﬁrst real health
plan migration to NewUM. The target completion, per the SOW and project plans, is June 2026.
This document is a product scoping guide for product, operational, and technical teams to
make Version 1.0 decisions. It is intentionally concise and does not capture all product details;
those will be managed with stakeholder input in Product Requirement Documents
(PRDs) and NewUM roadmap artifacts.
End- State C apabilities for Version 1.0
C ase Submission & Data Foundations
Providers and health plans can submit chemotherapy drug prior authorization cases through
current intake methods (fax, phone, and web).
Treatment details (schedules, dosing, frequency, etc.) are captured using an industry- standard
data structure with guardrails to improve accuracy and completeness.
The Admin Portal serves as the truth for all clinical data (drugs, regimens, diagnoses,
etc.) and health plan speciﬁc drug coverage policies (prior authorization status, non- preferred
drugs, etc.) and formulary ﬁle information for pharmacy drugs (quantity limits, step therapy).
Member eligibility and provider network data are sourced from the health plan's existing
inbound data ﬁles.
C linical Review & Determinations
OH Operational teams ( C RS, RNs, PharmDs, and MDs) can see complete protocol details within
a case for medical necessity review, document clinical notes, record any required clinical
interventions, and make ﬁnal drug level determinations. The health plan's speciﬁc drug
coverage policies are visible, actionable, and consistently applied within
their review workﬂows.

Summary Document
Overview
This document provides a concise summary of the infrastructure and services related to the
Letters v2 project, focusing on data management, APIs, and related technologies.
Data Management
The data management infrastructure includes various SQL Server databases such as EligibilityMS,
ProviderMS, DrugsMS, and LoggingDB. These databases support diﬀerent aspects of the project,
including eligibility, provider information, and drug data management. Additionally, there are
multiple Blob Storage solutions for storing inbound faxes and Matis documents, along with other
storage needs.
APIs and Services
Several APIs and services are integral to the system's functionality:
Eligibility API: Manages eligibility- related queries and data processing.
Provider API: Handles provider- related information and interactions.
Drugs API: Facilitates drug data management and retrieval.
Admin Portal API: Provides administrative functionalities and access control.
Mail Service (Kaleidoscope): Manages email communications.
IVR (Voice Recognition) API: Supports voice recognition capabilities.
Turnaround Time (TaT) API: Monitors and reports on processing times.
Data Processing and ETL
The system employs various ETL (Extract, Transform, Load) processes to manage data ﬂow and
transformation:
Azure Data Factory ETL and Databricks ETL: Facilitate data integration and processing.
Salesforce ETL: Integrates Salesforce data into the system.
Synapse ETL and SSIS ETL: Support data warehousing and integration tasks.
Additional Services
EventProcessor Windows Service: Manages event- driven processes.
MessageBridge Windows Service: Facilitates message routing and processing.
Fax Service API: Handles fax communications.
Document API: Manages document- related operations.
